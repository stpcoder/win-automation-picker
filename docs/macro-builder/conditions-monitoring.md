# 조건과 모니터링

## If와 Monitor의 차이

| 종류 | 실행 흐름 | 사용 예 |
| --- | --- | --- |
| `만약 ...` | 조건이 맞을 때만 안쪽 블록 실행 | PASS이면 다음 버튼 클릭 |
| `텍스트/색상 상태` | 클릭 없이 상태만 읽어 통과/실패 기록 | CH11의 현재 테스트 상태 확인 |

`모니터링 > 한 번 확인`과 `자동 시작`은 모니터 규칙만 평가합니다. 클릭, 입력, 키 블록은 실행하지 않습니다.

## 텍스트 조건

1. `텍스트 상태` 또는 `만약 텍스트라면`을 원하는 위치로 끌어 놓습니다.
2. 텍스트가 표시되는 component를 클릭합니다.
3. 오른쪽에서 조건을 선택합니다.

| 조건 | 의미 |
| --- | --- |
| `contains` | 일부 문자열 포함 |
| `equals` | 전체 문자열이 정확히 같음 |
| `starts_with` | 지정 문자열로 시작 |
| `ends_with` | 지정 문자열로 끝남 |
| `regex` | 정규식과 일치 |
| `not_empty` | 비어 있지 않음 |

4. `기대값`을 입력하고 `변경 적용`을 누릅니다.
5. 실제 화면 값으로 갱신하려면 `현재 값 읽기`를 누릅니다.

## 색상 조건

1. `색상 상태` 또는 `만약 색상이라면`을 끌어 놓습니다.
2. 상태 표시 component의 중심을 클릭합니다.
3. 캡처 시점의 색상이 `#RRGGBB` 기대값으로 자동 입력됩니다.
4. `색상 오차`를 조정합니다. 기본값은 `24`입니다.
5. 색이 바뀐 뒤 다시 샘플링하려면 `현재 값 읽기`를 누릅니다.

색상 비교는 RGB 거리 기준입니다. 안티앨리어싱이나 테마 차이로 픽셀이 조금 달라질 수 있으므로 정확히 같은 값만 요구하기보다 작은 오차를 둡니다.

## AND / OR 묶기

작업실에서 직접 묶는 방법:

1. `AND 조건 묶음` 또는 `OR 조건 묶음`을 놓습니다.
2. 텍스트/색상 조건 블록을 묶음 안으로 끌어 넣습니다.

모니터링 표에서 묶는 방법:

1. Ctrl 또는 Shift로 같은 위치의 규칙을 두 개 이상 선택합니다.
2. `선택 규칙 AND 묶기` 또는 `선택 규칙 OR 묶기`를 누릅니다.

AND 예시:

- 창 안 텍스트가 `CH11`을 포함함
- 상태등 색상이 초록색임
- 두 조건이 모두 맞으면 `CH11 / PASS`

OR 예시:

- 결과 텍스트가 `FAIL`
- 또는 상태등이 빨간색임
- 하나만 맞아도 `FAIL`

## 모니터 보드에 연결

규칙을 선택하고 오른쪽 또는 `모니터링` 탭에서 다음 값을 설정합니다.

| 필드 | 예시 | 의미 |
| --- | --- | --- |
| `탭` / `보드` | `SK Commander` | 사용자 지정 모니터 화면 이름 |
| `장비 / CH` | `CH9`, `CH11`, `PC04-RIG2` | 고정 형식이 아닌 자유 장비명 |
| `표시 상태` | `RUNNING`, `PASS`, `FAIL` | 이 규칙이 의미하는 상태 |

`CH1`, `CH11`처럼 이름이 겹쳐도 목록은 자유 입력값을 그대로 사용합니다.

## SK Commander 4개 창 예시

각 창마다 다음 규칙 묶음을 만듭니다.

1. `텍스트 상태`: CH 식별 component가 `CH9`와 정확히 일치
2. `색상 상태`: 상태 component가 파란색에 가까움, 상태 `RUNNING`
3. `색상 상태`: 같은 component가 초록색에 가까움, 상태 `PASS`
4. `색상 상태`: 같은 component가 빨간색에 가까움, 상태 `FAIL`
5. 필요한 CH 식별 조건과 상태 조건을 AND로 묶음

CH 목록은 `CH9, CH10, CH11, CH12`처럼 입력한 뒤 여러 규칙을 선택하고 `선택 규칙에 적용`을 누르면 순서대로 배정됩니다.

## 한 번 확인과 자동 확인

| 명령 | 동작 |
| --- | --- |
| `한 번 확인` | 모든 모니터 규칙을 1회 평가 |
| `자동 시작` | 입력한 주기마다 반복 평가 |
| `중지` | 다음 규칙 사이에서 안전하게 중단 |

표에는 `통과/실패`, 최근 읽은 값이 표시되고 위쪽에 `최근 확인 HH:MM:SS`가 갱신됩니다. AND/OR 묶음은 자식 조건을 한 번씩만 읽고 상세 결과를 각 자식 행에도 반영합니다.

## 보드 화면 커스터마이즈

`보드 화면 구성`에서 다음을 바꿀 수 있습니다.

- 화면 이름
- 행 기준: 장비/CH, 상태, 보드, 블록
- 열 기준: 상태, 장비/CH, 보드
- 보드 표시 순서
- 상태 표시 순서

규칙의 `탭` 값마다 미리보기 안에 실제 탭이 자동 생성됩니다. 예를 들어 `SK Commander`, `결과 텍스트`를 입력하면 두 화면을 따로 전환해 볼 수 있습니다.

`자동 구성`은 현재 규칙의 보드, CH, 상태 값을 읽어 기본 배치를 만듭니다.
