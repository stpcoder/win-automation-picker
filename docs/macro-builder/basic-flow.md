# 기본 매크로 만들기

## 가장 빠른 방법: 연속 녹화

클릭 블록과 입력 블록을 하나씩 따로 만들 필요가 없습니다.

1. `입력값을 PC별 변수로`를 켭니다. PC마다 같은 값을 입력할 때만 끕니다.
2. 필요하면 `동작 사이 대기 포함`을 켭니다. 0.6초 이상의 간격이 대기 블록으로 변환됩니다.
3. 상단 `연속 녹화 시작`을 누릅니다.
4. 대상 프로그램에서 평소처럼 입력칸을 클릭하고 타이핑한 뒤 버튼을 클릭합니다.
5. Win Automation Picker로 돌아와 `녹화 정지`를 누릅니다.
6. 정지 버튼 클릭은 제외되고 외부 프로그램에서 발생한 동작만 `녹화 타임라인`에 남습니다.
7. 타임라인과 가운데 작업실에 생성된 클릭, 입력, 키, 대기 블록을 확인합니다.

녹화 상태는 항상 상단에 `녹화 중 mm:ss · 동작 N개`로 표시됩니다. 녹화가 끝나면 각 입력칸의 최종 UIA 값을 한 개의 입력 블록으로 묶습니다. 따라서 한글 IME 입력이나 붙여넣기를 키 하나씩 분해하지 않습니다.

!!! warning "보안 입력"
    UIA가 비밀번호 필드로 표시한 component의 값은 workflow, 로그, FTP package에 저장하지 않습니다. 해당 블록은 빈 필수 변수로 생성되므로 PC별 실행표에서 런타임 값을 따로 지정해야 합니다.

## 입력값을 PC별로 바꾸기

연속 녹화에서 `입력값을 PC별 변수로`를 켜면 `Seq 1` 같은 실제 입력은 `${sequenceinput_value}` 형태로 바뀌고, 녹화값은 workflow의 기본값으로 저장됩니다.

- 타임라인에서 입력 행을 선택하고 `선택 입력을 PC별 변수로`를 누르면 나중에 변수로 전환할 수 있습니다.
- `선택 입력을 고정값으로`를 누르면 모든 PC가 같은 문자열을 사용합니다.
- 오른쪽 블록 설정에서도 `고정값 / PC별 변수`, 변수 이름, 기본값을 수정할 수 있습니다.
- 보안 입력은 고정값으로 바꿀 수 없습니다.

## 클릭 블록 하나만 만들기

1. 상단 `클릭 녹화`를 누릅니다.
2. 대상 프로그램에서 실제 버튼을 한 번 클릭합니다.
3. 가운데 작업실에 생긴 블록을 확인합니다.
4. 오른쪽 `이름`을 `시작 클릭`처럼 바꾸고 Enter를 누릅니다.
5. 캡처 품질 배지가 `정상`, `확인 필요`, `재검토` 중 무엇인지 확인합니다.

원하는 위치에 바로 넣는 방법:

1. 왼쪽 `이벤트 > 클릭 녹화` 블록을 원하는 블록 사이로 끌어 놓습니다.
2. 대상 프로그램의 버튼을 클릭합니다.
3. 새 클릭 블록이 끌어 놓았던 위치에 생성됩니다.

## 입력 블록 하나만 만들기

1. 상단 `입력값`에 실제 문자열 또는 `${message}` 같은 변수를 입력합니다.
2. 기존 내용을 지울 경우 `기존값 지우기`를 켭니다.
3. 기본 방식은 `paste`입니다. 붙여넣기를 막는 프로그램이면 `keys`로 바꿉니다.
4. `입력 녹화`를 누르고 대상 입력칸을 클릭합니다.
5. 오른쪽 `이름`을 `serial_input`처럼 바꿉니다.
6. 필요하면 오른쪽 `입력할 텍스트`에서 블록별 값을 수정하고 `변경 적용`을 누릅니다.

!!! info "수동 입력 녹화와 연속 녹화의 차이"
    `입력 녹화`는 입력칸 위치만 한 번 캡처합니다. `연속 녹화 시작`은 사용자가 실제로 타이핑한 최종 문자열까지 묶어서 기록합니다.

## 순서 바꾸기와 중첩

- 블록을 잡아 위아래로 끌면 실행 순서가 바뀝니다.
- `N번 반복`, `만약 ...` 블록의 빈 안쪽으로 다른 블록을 끌면 자식 블록이 됩니다.
- 자식 블록을 컨테이너 밖으로 끌어 다시 최상위 순서로 만들 수 있습니다.
- 드롭 가능한 위치에는 파란 삽입선이 표시됩니다.
- 블록을 더블클릭하면 오른쪽 이름 입력칸으로 바로 이동합니다.

## 실행

1회 실행:

1. 상단 `실행`을 누릅니다.
2. 비어 있는 반복/조건 블록이나 대상이 없는 블록이 있으면 실행 전에 해당 블록이 선택되고 오류 목록이 표시됩니다.
3. `실행 기록`에서 진행 순서와 오류를 확인합니다.

행별 실행:

1. `데이터 / 고급 > 데이터 행`에 Excel/Google Sheets 데이터를 붙여넣습니다.
2. 첫 행이 열 이름이면 `첫 행을 열 이름으로 사용`을 켭니다.
3. 상단 `데이터 실행`을 누릅니다.

## 저장과 복구

| 작업 | 방법 |
| --- | --- |
| 저장 | 상단 `저장` |
| 다른 파일로 저장 | 상단 `더보기 > 다른 이름으로 저장` |
| 불러오기 | 상단 `불러오기` |
| Python 생성 | 상단 `Python 내보내기` |
| 직전 편집 취소 | 작업실 `되돌리기` 또는 `Ctrl+Z` |
| 취소한 편집 복구 | 작업실 `다시 실행` 또는 `Ctrl+Y` |
| 블록 복제 | 오른쪽 `복제` 또는 `Ctrl+D` |
| 블록 삭제 | 오른쪽 `삭제` 또는 Delete |
| 컨테이너만 제거 | 오른쪽 `풀기`; 내부 블록은 유지 |

프로젝트 JSON에는 블록, 모니터 화면 구성, PC별 입력 변수 기본값, 데이터 행, 첫 행 헤더 옵션과 행 간 대기값이 함께 저장됩니다. 기존 recipe JSON도 그대로 불러올 수 있습니다. 저장하지 않은 변경이 있으면 종료 전에 저장 여부를 확인합니다.

## 잘못 캡처했을 때

1. 잘못된 블록을 선택합니다.
2. 캡처 품질 줄의 `선택 대상 다시 잡기` 또는 오른쪽 같은 이름의 버튼을 누릅니다.
3. 올바른 component를 다시 클릭합니다.
4. `선택 블록 시험`으로 한 블록만 확인합니다.
5. 같은 프로그램 창이 여러 개라면 `대상 고급 설정`에서 창 구분 조건을 추가합니다.
