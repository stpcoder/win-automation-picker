# 설정 파일과 폴더

일반 작업자는 설정 파일을 직접 편집하지 않습니다. 설정 파일 분실 또는 담당자 점검 시 아래 내용을 확인합니다.

## 기본 파일

| 파일 | 생성 위치 | 내용 |
|---|---|---|
| `fixture-connection.info` | 관리자 PC와 실장기 PC | 통신 서버, 관리자 PC, 실장기 PC, 실장기 정보 |
| `fixture-device.config.json` | 실장기 PC | COM, Baud rate, 전원·Binary 도구 설정 |
| `*.aework.json` | 관리자 PC | SEQ와 자동 실행 순서를 묶은 작업 파일 |
| `*.macro.json` | 자동 실행 순서 편집기 | 클릭, 입력, 반복, 조건, 상태 확인 규칙 |
| `*.py` | 자동 실행 순서 편집기 | 실행 가능한 Python 파일 |
| `*.fixtureseq.zip` | SEQ 검사 결과 | 검사한 SEQ와 실행 정보 |
| `*.fixturebinary.json` | Binary 검사 결과 | Binary 이름, XML, 파일 검사값 |

## 비밀번호 저장 방식

통신 서버 비밀번호를 설정 파일에 직접 적지 않아야 하는 경우 Windows 환경 변수를 사용합니다.

1. `3 초기 설정 > 통신 설정`을 엽니다.
2. `비밀번호 환경 변수`에 환경 변수 이름을 입력합니다. 예: `AE_COMM_PASSWORD`.
3. Windows에서 같은 이름의 사용자 환경 변수를 만듭니다.
4. 프로그램을 다시 실행하고 `연결 확인`을 누릅니다.

설명서, 화면 캡처, 로그에 실제 비밀번호를 남기지 않습니다.

## 통신 서버 전용 폴더

프로그램은 설정한 전용 폴더 아래만 사용합니다.

```text
mobile-dram-ae/
├─ packages/       테스트에 사용할 파일
├─ jobs/           실장기 PC가 확인할 요청
├─ status/         실장기 PC 상태
├─ results/        테스트 결과
├─ screenshots/    요청한 전체 화면
├─ metadata/       Binary와 장착 자재 정보
└─ stops/          긴급 중단 요청
```

다른 프로그램이 사용하는 서버 폴더를 전용 폴더로 지정하지 않습니다. `mobile-dram-ae`처럼 이 프로그램만 사용하는 새 폴더를 지정합니다.

## 실장기 PC 시작 폴더

`시작 폴더 만들기`를 실행하면 PC별 설정이 나뉩니다.

```text
TFT30-1/
├─ AEWorkbench.exe
├─ fixture-connection.info
├─ fixture-device.config.json
└─ README-SETUP.txt
```

`TFT30-1` 폴더에는 다른 PC 정보가 들어가지 않습니다. 서로 다른 실장기 PC의 설정 파일을 한 폴더에 섞지 않습니다.

## 설정 파일 재생성

설정 파일을 잃어버렸다면 관리자 PC에서 다시 만듭니다.

1. 관리자 PC에서 최신 Binary와 자재 정보가 들어왔는지 확인합니다.
2. `3 초기 설정 > 실장기 PC 목록`을 엽니다.
3. 대상 PC를 선택합니다.
4. `시작 폴더 만들기`를 다시 실행합니다.
5. 기존 실장기 PC 폴더에 덮어쓰기 전에 현장 수정 정보가 최신인지 확인합니다.

## 직접 편집 확인 항목

담당자가 JSON을 직접 수정한 경우 프로그램에서 반드시 다시 확인합니다.

- 같은 내부 식별값이 두 PC에 사용되지 않았는지
- 한 PC에 실장기가 4대를 넘지 않는지
- 같은 COM이 두 실장기에 중복되지 않았는지
- 통신 서버 전용 폴더가 비어 있지 않은지
- Windows 이름이 실제 `hostname`과 같은지
- Binary와 기본 정보 수정 시각 형식이 올바른지

`연결 구조 > 구성 검사`에서 `진행 불가`가 없어야 합니다.
