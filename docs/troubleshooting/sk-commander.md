# SK Commander 인식

## 다른 창의 값을 읽음

같은 SK Commander 창이 여러 개면 창 제목만으로 구별하지 못할 수 있습니다.

1. 각 창의 실장기 번호가 실제 장비와 맞는지 확인합니다.
2. 실장기 번호 텍스트 규칙의 기대값을 확인합니다.
3. 창 구분 조건과 실장기 번호 조건을 AND로 묶습니다.
4. 규칙의 실장기 선택값이 `CH1`처럼 한 대만 지정되어 있는지 확인합니다.
5. 다른 창을 앞으로 가져온 뒤 `현재 값 읽기`를 다시 실행합니다.

## 빈 영역이나 큰 패널이 선택됨

1. 대상을 다시 선택합니다.
2. 마우스를 글자나 색상 표시의 가운데에 놓습니다.
3. `캡처 진단`에서 선택한 Control type과 대상 경로를 확인합니다.
4. 텍스트가 직접 읽히지 않으면 상위 칸의 텍스트를 확인하거나 색상 규칙을 사용합니다.
5. 클릭 블록이라면 `선택 블록 테스트`로 실제 버튼 반응을 확인합니다.

선택할 때 화면에 강조 표시가 보였다는 이유만으로 정상이라고 판단하지 않습니다. 반드시 읽은 값이나 클릭 결과까지 확인합니다.

## 색상 상태가 자주 바뀜

- Windows 화면 배율을 초기 연결 때와 같게 유지합니다.
- 색상 기준을 실제 상태마다 다시 읽습니다.
- 색상 오차를 너무 크게 설정하지 않습니다.
- 파랑, 초록, 빨강 규칙이 같은 실장기와 같은 화면 부분을 보는지 확인합니다.
- 가능하면 색상 규칙과 PASS/FAIL 텍스트 규칙을 AND 또는 OR로 결합합니다.

## SoC 또는 자재가 비어 있음

1. 해당 텍스트 규칙의 `SK Commander에서 읽을 항목`이 맞는지 확인합니다.
2. SoC는 `SoC`, 자재는 `장착 자재 ID`를 선택합니다.
3. 실장기 선택값이 비어 있지 않은지 확인합니다.
4. 실장기 PC에서 `테스트 모니터링 시작`이 켜져 있는지 확인합니다.
5. 관리자 PC에서 상태를 새로고침합니다.

## BL1, BL2, LK, OS가 맞지 않음

부팅 단계 규칙은 단계별 표식을 읽어야 합니다. Console에 `LK2]`처럼 표시되는 경우에는 해당 텍스트가 포함되는지 확인하는 규칙을 만들고 읽을 항목을 `부팅 단계 (BL1/BL2/LK/OS)`로 지정합니다. 여러 단계가 동시에 화면에 남으면 가장 최근에 표시되는 영역을 대상으로 선택합니다.
