# 통신과 실장기 PC

## 실장기 PC가 목록에 나타나지 않음

1. 실장기 PC에서 `AEWorkbench.exe`가 실행 중인지 확인합니다.
2. `3 초기 설정 > 이 실장기 PC`에서 `통신 시작`을 누릅니다.
3. `이 실장기 PC 식별값`이 관리자 PC에 등록한 이름과 같은지 확인합니다.
4. `fixture-connection.info`가 실행 파일과 같은 폴더에 있는지 확인합니다.
5. 관리자 PC와 실장기 PC에서 각각 `연결 확인`을 누릅니다.
6. 서버 주소, 포트, 아이디, 서버 폴더가 양쪽에서 같은지 확인합니다.

## 연결 확인은 되지만 상태가 오래됨

- `조회 간격`은 기본 5초입니다.
- PC마다 `조회 분산`이 적용되므로 동시에 요청하지 않습니다.
- Windows 시간이 맞는지 확인합니다.
- 실장기 PC의 통신 기록에서 `다시 연결 중`이 반복되는지 확인합니다.
- 관리자 PC에서 `새로고침`을 눌러 `마지막 상태 조회` 시각을 확인합니다.

## 다른 서비스에 영향을 줄까 걱정되는 경우

프로그램은 조회 또는 파일 전송마다 연결을 열고 작업이 끝나면 닫습니다. 한 연결을 계속 점유하지 않습니다. 다음 기본값으로 서버 부하를 제한합니다.

- 실장기 PC마다 조회 시점을 무작위로 분산
- 화면은 요청할 때만 한 번 전송
- 오류 화면 전송 최소 간격 적용
- 결과, 로그, 화면, 실행 파일의 보관 개수 제한
- 임시 파일에 올린 뒤 완료 시 이름을 바꾸어 불완전한 파일 노출 방지

서버 사용량이 높으면 `고급 정책`에서 조회 간격을 늘리고, 보관 개수를 줄이고, 자동 화면 저장을 끕니다.

## 전체 화면이 늦게 열림

화면 요청 시간은 다음 합계입니다.

1. 실장기 PC의 다음 조회까지 기다리는 시간
2. 화면 저장 시간
3. 파일 전송 시간
4. 관리자 PC가 결과를 확인하는 시간

기본 조회 간격이 5초라면 즉시 표시되지 않을 수 있습니다. 같은 PC에 연속 요청하지 말고 `마지막 조회 시각`을 확인합니다.

## 설정 파일을 잃어버림

관리자 PC의 `실장기 PC 목록`에서 대상을 선택하고 `시작 폴더 만들기`를 다시 실행합니다. 기존 실장기 PC 폴더에 덮어쓰기 전에는 현장 Binary와 자재 정보가 관리자 PC에 최신으로 들어왔는지 먼저 확인합니다.
