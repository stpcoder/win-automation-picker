# 상태 확인 상세

상태 확인은 테스트를 진행하는 동안 필요한 정보만 읽는 기능입니다. 평상시에는 `새로고침`으로 한 번 확인하고, 테스트 중에만 연속 모니터링을 사용합니다.

## 상태 화면 구분

| 화면 | 주로 보는 값 |
|---|---|
| 테스트 상태 | 테스트 이름, 실행 ID, 진행·PASS·FAIL·중지, Grid 진행 수, 최근 결과 |
| 실장기 상태 | 실장기 PC, 실장기 번호, SoC, Binary, DRAM, 자재 ID, 부팅 단계, 고장 상태 |
| 모니터 보드 | 사용자가 만든 탭별 텍스트·색상 조건 결과 |

## 새로고침과 연속 모니터링

- `새로고침`: 지금 한 번만 최신 상태와 선택한 결과를 읽습니다.
- `모니터링 시작`: 테스트 중 일정 간격으로 상태를 읽습니다.
- `모니터링 중지`: 연속 확인을 즉시 끝냅니다.
- `최근 불러온 시각`: 화면에 표시된 정보가 언제 읽힌 것인지 보여줍니다.

진행 중인 테스트가 없으면 연속 모니터링은 자동으로 종료됩니다. 상태가 오래된 것처럼 보이면 먼저 최근 불러온 시각을 확인한 뒤 새로고침합니다.

## 사용자 지정 모니터 보드

자동 실행 순서에서 상태 블록마다 다음 값을 지정할 수 있습니다.

1. 탭 이름: 예 `SK Commander`, `부팅 단계`, `결과 텍스트`
2. 실장기: `${channel}` 또는 `CH3`
3. 표시 상태: `진행 중`, `PASS`, `FAIL`, `LK` 등
4. 읽을 항목: 일반 상태, 실장기 번호, SoC, 자재, 테스트 이름, 테스트 상태, 부팅 단계
5. 판정 조건: 텍스트 또는 색상과 허용 오차

하나의 탭에 여러 실장기의 같은 항목을 배치하거나, 한 실장기의 여러 조건을 묶을 수 있습니다.

### SK Commander 네 창을 구별하는 예

CH3의 PASS를 판정하려면 다음 두 조건을 AND로 묶습니다.

1. `실장기 번호` 항목의 텍스트가 `${channel}`과 같음
2. `테스트 상태` 항목의 색상이 초록색 범위임

FAIL은 다음 조건을 OR로 묶을 수 있습니다.

1. 상태 색상이 빨강색 범위임
2. 결과 텍스트에 `FAIL`이 포함됨

창 제목만으로 구별하지 않고, 창 안의 실장기 번호를 함께 비교해야 CH9, CH10, CH11, CH12처럼 두 자리 번호도 정확히 구별할 수 있습니다.

## 색상 판정 설정

1. 실제 테스트 화면에서 판정할 항목을 선택합니다.
2. `현재 값 읽기`를 눌러 RGB 값을 가져옵니다.
3. 파랑, 초록, 빨강 상태를 각각 다른 조건으로 저장합니다.
4. 화면 밝기와 Windows 배율 차이를 고려해 작은 허용 오차를 줍니다.
5. 정상 상태와 경계 상태를 각각 `선택 블록 테스트`로 확인합니다.

상태 표시 내부의 가운데 지점을 선택합니다. 글자와 테두리는 선택하지 않습니다. 투명 효과나 움직이는 강조 표시가 있으면 텍스트 조건을 함께 사용합니다.

## 텍스트 판정 설정

| 조건 | 예 |
|---|---|
| 같음 | 부팅 단계가 정확히 `LK` |
| 포함 | 결과에 `PASS` 포함 |
| 시작 | Grid 이름이 `RH_`로 시작 |
| 반전 | `FAIL`이 포함되지 않음 |

공백, 줄바꿈, 대소문자가 바뀌는 항목은 `같음`보다 `포함`을 사용합니다. 숫자를 비교해야 하면 화면에 표시된 단위까지 포함해 기록합니다.

## Grid 진행 확인

SEQ의 `#이름`은 테스트 구간과 로그 저장 단위로 사용합니다. 상태 화면에서는 다음을 확인합니다.

- 전체 Grid 수
- 완료한 Grid 수
- 현재 Grid 이름
- 마지막 완료 시각
- 실패한 Grid와 관련 로그

예: `5 / 12 · GRID_RH_HT_HV 실행 중`. Grid 수가 늘지 않는데 최근 연락 시각은 계속 바뀐다면 SK Commander 로그와 부팅 단계를 확인합니다.

## 전체 화면 요청

전체 화면은 상시 전송하지 않습니다.

1. 확인할 실장기 PC를 선택합니다.
2. `전체 화면 보기`를 누릅니다.
3. 새 요청 ID와 요청 시각이 표시되는지 확인합니다.
4. 실장기 PC가 한 번 캡처해 통신 서버에 올릴 때까지 기다립니다.
5. 같은 요청 ID의 최신 화면이 도착하면 표시됩니다.

화면은 원본 해상도로 저장하고 문서나 창 크기에 맞춰 표시만 축소합니다. 오래된 화면을 최신 화면으로 오인하지 않도록 요청 시각과 불러온 시각을 함께 봅니다.

## 저장 공간과 통신 부하

- 상태는 작은 구조화 파일로 보내며 매번 전체 로그를 올리지 않습니다.
- 화면은 요청받았을 때 한 번만 생성합니다.
- 로그와 화면은 설정한 보관 개수를 넘으면 오래된 파일부터 정리합니다.
- 연결 실패 시 짧은 간격으로 무한 재시도하지 않고 대기 시간을 늘립니다.
- 다른 프로그램이 같은 통신 서버를 사용하더라도 전용 폴더 밖의 파일은 읽거나 지우지 않습니다.

## Excel 저장

`Excel 내보내기`는 현재 화면에 불러온 시점의 실장기 PC와 실장기 상태를 저장합니다. 파일에는 최근 불러온 시각을 포함합니다.

내보내기 전에 한 번 새로고침하고, 오래된 상태가 있는 행은 그대로 보고될 수 있으므로 최근 연락 시각을 함께 확인합니다.
