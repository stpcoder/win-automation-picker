# SEQ와 자동 실행 순서 준비

## 1. SEQ 준비

1. `2 SEQ · 자동 실행 순서 준비 > SEQ · 자동 실행 순서`를 엽니다.
2. `SEQ 작성 파일`에서 작성한 파일을 선택합니다.
3. `검사 완료 SEQ`의 저장 위치를 확인합니다.
4. `오류 검사 · 내보내기`를 누릅니다.
5. 결과가 `SEQ PASS`인지 확인합니다.

SEQ 명령은 `;`로 구분합니다. 한 줄 형식에서 `;` 다음에 불필요한 공백이 들어가면 대상 프로그램에서 오류가 날 수 있으므로 검사 결과를 반드시 확인합니다.

```text
log 0xff;tm -de 0x2;exit;exit
```

부팅 단계 전환, `clk.sh`, 온도, VDD, 4-corner 조건은 작성한 SEQ 목적에 맞게 포함합니다.

## 2. 자동 실행 순서 녹화

1. `순서 편집`을 누릅니다.
2. `연속 녹화 시작`을 누릅니다.
3. SK Commander에서 평소처럼 Load, 입력, Start 순서로 조작합니다.
4. 편집 화면으로 돌아와 `녹화 정지`를 누릅니다.
5. 아래 녹화 목록에서 프로그램, 동작, 대상, 입력값을 확인합니다.

편집 프로그램 안에서 누른 `녹화 정지`는 기록되지 않습니다. 외부 프로그램에서 수행한 클릭과 입력만 블록으로 만들어집니다.

## 3. 블록 편집

- 블록 이름을 `SEQ Load`, `자재 ID 입력`, `테스트 시작`처럼 바꿉니다.
- 블록을 잡아 위아래로 이동합니다.
- 반복 블록 안으로 끌어 넣어 반복 횟수를 지정합니다.
- 텍스트 또는 색상 조건 블록 안에 실행할 블록을 넣습니다.
- 필요 없는 블록은 선택 후 Delete 키 또는 삭제 버튼으로 지웁니다.
- 실행 취소와 다시 실행으로 이동 결과를 되돌릴 수 있습니다.

## 4. 실장기별 입력값 만들기

녹화한 텍스트 입력은 두 방식 중 하나로 설정합니다.

| 방식 | 사용 시점 | 예 |
|---|---|---|
| 고정값 | 모든 실장기에 같은 값 입력 | SEQ 이름 |
| 실장기별 입력값 | 실장기마다 값이 다름 | 장착 자재 ID, Slot, 온도 이름 |

텍스트를 보내는 방법은 기본값인 `한 번에 입력`을 사용합니다. 대상 프로그램에서 일부 글자가 빠지면 해당 블록을 선택하고 `한 글자씩 입력`으로 바꿔 다시 시험합니다.

녹화 목록에서 입력 동작을 선택하고 `선택 입력을 실장기별 값으로`를 누릅니다. 변수 이름은 `material_id`, `sequence_name`, `temperature_c`처럼 지정합니다.

실행할 때 `TFT30-1 / CH1`에는 `AA-1`, `TFT30-1 / CH2`에는 `AA-2`를 넣을 수 있습니다. 자동 실행 순서의 텍스트에는 `${material_id}`가 사용되고 실제 값은 실행 직전에 각 행에서 들어갑니다.

같은 SEQ를 여러 TFT/UTF에 보내는 경우에도 `sequence_name`만 같게 두고 `material_id`는 실장기 정보에서 불러온 값을 유지합니다. 공통 입력값에 `AA-1`을 한 번 적어 모든 실장기에 덮어쓰지 않습니다.

## 5. 테스트와 Python 준비

1. `값 편집`에서 이 PC에서 테스트할 임시 값을 입력합니다.
2. `테스트 실행`을 눌러 선택한 프로그램에서 실제 동작을 확인합니다.
3. 이상이 있으면 즉시 `중지`를 누릅니다.
4. `검사 · Python 준비`를 눌러 실행 가능한 `.py` 파일을 만듭니다.
5. `통신 서버에 등록`을 누릅니다.

Python 파일은 블록 순서, 반복, 조건, 실장기별 입력값을 포함하므로 별도 편집 없이 실행할 수 있습니다.
